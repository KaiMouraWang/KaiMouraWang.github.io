INDEX reading rule in String/List‚Ä¶

In index, Python always read from Left to Right, even it is from a positive integer from a negative integer, it will NEVER go from right to left. Remember, in index, positive index(include 0), negative index are just the same reference using different numbers.

list = ['a', 'b', 'c', 'd', 'e', 'f', 'g', 'h']
#a reference has two index-numbers AND they're exchangable
#list_index = [0, 1, 2, 3, 4, 5, 6, 7] same as:
#list_index = [-8, -7, -6, -5, -4, -3, -2, -1]
#for "d", its index can be 3 or -5, they both stand for "d", it's like a "label", NOT a math-related calculation
print(list[2:-2]) #1
#is the SAME as
print(list[-6:6]) #2
#is the SAME as
print(list[2:6]) #3
#is the SAME as
print(list[-6:-2]) #4

#The natural or negative index is just a "label"! 
#List[a,b]will ALWAYS read from Left to Right!
#You can MIX them however you want!
#As long as they stand for the same reference!
#Actually there are 4 different ways to say the same for every list[a,b]!

Why INDEX starts with 0 ?

0-based languages : JavaScript, Python, Java, PHP, Ruby, or Go: where the first element is at index 0;

In R, MATLAB, Julia: array indexes start at 1 - the 1st element is at index 1.

The most common answer to the array numbering question, points out that zero-based numbering comes from language design itself. In C language, an array points to the location in the memory, so in expression array[n], n should not be treated as an index, but as an offset from the array‚Äôs head.

To illustrate that, we can run the C program below:

#include <stdio.h>int main()
{
    int data[6] = {1, 2, 3, 4, 5, 6};
    int i = 0;
  
    printf("Array address: %p\n", data);
  
    do {
        printf("Array[%u] = %p\n", i, (void *)(&data[i]));
        i++;
    } while(i < 6);
}

Output:

Array address: 0x7ffe9472bad0
Array[0] = 0x7ffe9472bad0
Array[1] = 0x7ffe9472bad4
Array[2] = 0x7ffe9472bad8
Array[3] = 0x7ffe9472badc
Array[4] = 0x7ffe9472bae0
Array[5] = 0x7ffe9472bae4

As we can see on this example, the first element-Array[0] and the array itself-Array address points tothe same memory location, so it is 0 elements away from the location of the array itself.

Similar 0-based: range function.

range(start, stop[, step])

The arguments to the range constructor must be integers (either built-in int or any object that implements the __index__ special method).

If the start argument is omitted, it defaults to 0.

Stop means < stop , ‚Äústop‚Äù before the number given at stop , do not include the number.

If the step argument is omitted, it defaults to 1. If step is zero, ValueError is raised.

If if and elif are put together, thenif elif (else) become a whole package. For example, in a block, both if and elif is True , then only if sentence will be executed and elif and elsewill be ignored. So basically this follows the reading rule of from left to right and from up to down.

    print("It's a lovely day.")

How to understand ‚Äòwhile ‚Ä¶ :‚Äô ‚Äòif ‚Ä¶ :‚Äô and ‚Äònot‚Äô

and, or, not

In if or while statement, if you don‚Äôt put not, the Boolean value of the condition will always be True, no matter what Boolean value you set to the condition before.

Example:

a = False # this is only saying a's initial state of Boolean value is False

while a: # means == while a (is True), even a's initial Boolean is False, doesn't mean here saying while a is False

if a: # means == if a (is True)

#when use 'not' in front of a condition,this means the while/if the condition is False, no matter what initial Boolean value of the condition

a = False
while not a: # means == while a (is False) Simple as that. Wrong thought would be False(a)+False(not)=True, it just doesn't work like that! Don't make it complicated!

If you want the condition to be False, then you should add not in front of the condition.

The use of.append( ) and += in List :

If you want to add numbers to a list, and you want to treat this list as a sequence of number, not string, you should use .append() method.

However, if you want to add a string or you only want to add a number as a string to the list, append( ) will not work, as it only takes number in the parenthesis. In such case, you should use += "<srting>" to add the string to the list.

list = [0, 1, 2, 3, 4, 5]

list.append(6) #add as a number
list += "6"      #add as a string
print(list)

list1 = ["hello", "world"]

list1 += "6"
print(list1)

#output:
>[0, 1, 2, 3, 4, 5, 6, '6'] #Notice here the last 6 is a string cause you use += "" to add it 
>['hello', 'world', '6']
